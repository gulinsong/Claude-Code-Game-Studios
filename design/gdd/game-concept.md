# Game Concept: 岁时记 (Suìshí Jì)

*Created: 2026-03-24*
*Status: Draft*

---

## Elevator Pitch

> 在一个被遗忘的古村落里，你成为新任「节令官」，负责筹备和主持四季的传统节日。探索村庄周边的神秘之地，与村民建立深厚的情谊，用你的双手让这个村庄重新热闹起来。
>
> 像《动物森友会》的陪伴感 + 中国传统节日的文化深度 + 微信小程序的随时随地。

## Core Fantasy

成为传统节日的守护者，亲手复兴一个消逝的民间文化世界。

你不是一个拯救世界的英雄，而是一个平凡却重要的角色——负责让村庄的四季节日重新焕发生机。你的力量不是战斗，而是筹备、制作、陪伴和传承。

在微信小程序的平台上，《岁时记》提供的是一个可以随时随地进入的、感受时间流转、体会人与人之间温暖的空间。等公交的时候、午休的时候、睡前的时候——打开就能进入那个温馨的村庄。

---

## Unique Hook

**"像《旅行青蛙》的陪伴感 + 以中国传统节日为核心玩法"**

- **节日是核心循环**，不只是背景装饰。每个节日都是一个小章节，有准备期、高潮和余韵。
- **手工艺有仪式感**——包粽子、做月饼、扎灯笼不是简单点击，而是有节奏、有操作的迷你游戏。
- **微信原生社交**——好友可以互访村庄、互相送礼、分享收集进度。
- **随时随地**——小程序即开即玩，云存档跨设备同步。

## Player Experience Analysis (MDA Framework)

### Target Aesthetics (What the player FEELS)

| Aesthetic | Priority | How We Deliver It |
| ---- | ---- | ---- |
| **Discovery** | 1 (Primary) | 探索村庄周边、收集民间传说、解锁新食谱和装饰 |
| **Fellowship** | 2 | 与村民建立友谊、好友互访送礼、感受人情味 |
| **Submission** | 2 | 没有时间压力、没有失败状态、温馨陪伴型体验 |
| **Fantasy** | 3 | 成为"节令官"、生活在古村落、体验传统文化 |
| **Narrative** | 3 | 每个节日有故事线、村民有各自的故事 |
| **Expression** | 4 | 装饰村庄、角色服装、节日布置风格 |
| **Sensation** | 4 | 四季视觉变化、节日氛围、手工艺的触感反馈 |
| **Challenge** | N/A | 无硬核挑战，游戏主打放松体验 |

### Key Dynamics (Emergent player behaviors)

- 玩家会利用碎片时间登录，查看村庄状态、完成小任务
- 玩家会主动探索地图，寻找隐藏的收集物
- 玩家会期待下一个节日的到来，提前准备材料
- 玩家会与微信好友分享自己的村庄、互相送礼
- 玩家会为了解锁新装饰而愿意付费

### Core Mechanics (Systems we build)

1. **节日筹备系统** — 按农历节奏准备节日，每个节日有独特的准备活动和庆典
2. **手工艺系统** — 制作节日物品的迷你游戏，触屏操作，有仪式感但不太难
3. **村民关系系统** — 对话、送礼、触发剧情，深化与村民的情谊
4. **探索收集系统** — 探索村庄周边，收集食谱、传说、装饰物
5. **村庄发展系统** — 节日后村庄会有变化，新区域开放，新村民到来
6. **社交系统** — 微信好友互访、送礼、分享
7. **内购系统** — 装饰、服装、体力/加速道具

## Player Motivation Profile

### Primary Psychological Needs Served

| Need | How This Game Satisfies It | Strength |
| ---- | ---- | ---- |
| **Autonomy** | 选择先准备哪个节日、怎么装饰村庄、和谁交朋友 | Core |
| **Competence** | 手艺熟练度提升、收集越来越完整、村庄越来越美 | Supporting |
| **Relatedness** | 与村民的情谊、与微信好友的互动 | Core |

### Player Type Appeal (Bartle Taxonomy)

- [x] **Explorers** (发现、理解系统、寻找秘密) — How: 探索地图、收集传说、发现隐藏内容
- [x] **Socializers** (关系、合作、社区) — How: 与村民互动、好友互访、微信分享
- [ ] **Achievers** (目标完成、收集、进度) — How: 收集系统有一定吸引力
- [ ] **Killers/Competitors** (主导、PvP、排行榜) — How: 不适用

### Flow State Design

- **Onboarding curve**: 第一个节日（中秋）作为教程，5分钟内让玩家理解核心循环
- **Difficulty scaling**: 没有难度曲线，手工艺熟练度提升效率
- **Feedback clarity**: 节日庆典视觉反馈、村民感谢、村庄变化、好友互动通知
- **Recovery from failure**: 无失败状态，节日准备不完美也能庆祝

## Core Loop

### Moment-to-Moment (30 seconds)

- **点击移动**到村庄和周边地点
- **点击采集**材料（花草、竹子、纸张、食材）
- **点击村民**对话，了解需求和故事
- **触屏手势**制作手工艺（滑动、点击、拖拽）

操作简单直观，单手可玩，适合碎片时间。

### Short-Term (5-15 minutes)

**"日"的循环**（可以加速或等待）：
- **早晨**：查看今日待办、收取产出
- **白天**：探索、采集、制作（消耗体力）
- **傍晚**：与村民互动、送礼物
- **夜晚**：整理收获、查看日记

体力系统控制单次游玩时长，体力耗尽可以等待恢复或使用道具。

### Session-Level (10-30 minutes)

围绕**小目标**的完整体验：
- 完成一个村民的请求
- 准备节日的一个部分
- 探索完一个小区域
- 与好友互动（互访、送礼）

每次退出都有"完成了什么"的满足感，同时有"下次再来"的期待。

### Long-Term Progression

| 成长类型 | 具体内容 |
|---------|---------|
| **村庄发展** | 新建筑解锁、新区域开放、新村民搬来 |
| **个人技能** | 学会新的手艺、食谱、装饰样式 |
| **故事收集** | 民间传说、村庄历史、村民秘密 |
| **关系深化** | 与村民的友谊加深，解锁更多互动 |
| **装扮收集** | 角色服装、村庄装饰（部分付费） |

**主线目标**：完成四季的节日循环，让村庄举办"岁末大典"

**无尽模式**：主线结束后，节日每年循环，可继续收集和深化关系

### Retention Hooks

- **Curiosity**: 下一个节日是什么？村民A的秘密？锁着的区域？
- **Investment**: 与村民的情谊、收集的传说食谱、精心装饰的村庄
- **Social**: 好友互访、微信分享、排行榜（收集度）
- **Mastery**: 手工艺熟练度、发现所有隐藏内容
- **Daily**: 每日登录奖励、限时活动

## Game Pillars

### Pillar 1: 人情味 (Rénqíngwèi)

每一个互动都应该让玩家感受到人与人之间的温度。

*Design test*: 如果某个村民对话感觉像"任务发布器"，重写它。村民会记得你做过的事、会主动找你聊天、会有自己的喜怒哀乐。

### Pillar 2: 四季流转 (Sìjì Liúzhuǎn)

时间不是数字，是可感知的变化——花开、叶落、雪飘。

*Design test*: 如果某个季节看起来和上个季节没区别，加更多细节。同一地点四季不同、节日氛围自然过渡、物产随季节变化。

### Pillar 3: 手艺的温度 (Shǒuyì de Wēndù)

制作不是点击按钮，是一次小小的仪式——有动作、有节奏、有成果。

*Design test*: 如果制作可以完全自动跳过，那设计失败了。包粽子要折叠、扎绳；做灯笼要剪裁、粘贴；每种手艺有独特触屏操作。

### Pillar 4: 发现民间之美 (Fāxiàn Mínjiān zhī Měi)

每一个收集物背后都有故事——不只是"东西"，是文化。

*Design test*: 如果某个食谱/传说没有背景介绍，补上它。食谱有来源故事、传说有口述版本、装饰有地域特色。

### Anti-Pillars (What This Game Is NOT)

- **NOT 农场模拟**: 重点是节日和人际，不是种田。
- **NOT 恋爱模拟**: 有友情和温暖，但不以恋爱为主要系统。
- **NO 失败状态**: 节日准备不完美也能庆祝，没有"游戏结束"。
- **NO 战斗系统**: 冲突通过对话和选择解决，不是打怪。
- **NO 强迫社交**: 社交是加分项，单人也能完整体验。
- **NO P2W内购**: 付费内容只影响外观和便利，不影响核心进度。

## Monetization Design

### 内购项目设计

| 类型 | 内容 | 价格区间 | 说明 |
| ---- | ---- | ------- | ---- |
| **装饰** | 村庄装饰、节日限定装饰 | ¥1-6 | 纯外观，不影响进度 |
| **服装** | 角色服装、配饰 | ¥6-18 | 纯外观 |
| **体力** | 体力恢复道具 | ¥1-3 | 便利性，可等待恢复 |
| **加速** | 缩短制作/采集时间 | ¥1-3 | 便利性 |
| **礼包** | 新手礼包、节日礼包 | ¥6-30 | 组合优惠 |
| **去广告** | 去除激励视频广告 | ¥18-30 | 一次性购买 |

### 广告设计

- **激励视频**：观看后获得体力/材料/金币（可选，非强迫）
- **无Banner广告**：保持沉浸感

### 付费原则

- 核心体验免费，付费只影响外观和便利
- 不设计P2W内容，不设计抽卡
- 尊重玩家时间，不刻意拖慢进度逼氪

## Technical Considerations

| Consideration | Assessment |
| ---- | ---- |
| **Recommended Engine** | Cocos Creator — 小程序支持最佳、中文文档、成熟生态 |
| **Key Technical Challenges** | 小程序包体限制、云存档同步、微信SDK集成 |
| **Art Style** | 2D简约插画风或像素风，中国传统美学，优化包体 |
| **Art Pipeline Complexity** | 中等 — 使用素材包 + AI辅助 + 简化风格，严格控制资源大小 |
| **Audio Needs** | 中等 — 中国风BGM、环境音效、手工艺音效（压缩） |
| **Networking** | 微信登录、云存档、好友系统 |
| **Content Volume** | 4个节日、8-10个村民、5-8个地点、4-6种手工艺、30-50个收集物 |
| **Package Size** | 主包 < 4MB，分包总计 < 20MB |

### 小程序技术方案

- **分包加载**：按季节/节日分包，降低首包大小
- **资源远程加载**：大图、音频走CDN
- **云开发**：使用微信云开发实现存档和社交功能
- **离线模式**：基础功能可离线，联网同步

## Risks and Open Questions

### Design Risks

- **碎片化体验可能削弱沉浸感** — 需要在短session内提供满足感
- **内购设计可能影响口碑** — 需要谨慎平衡，避免P2W印象
- **节日循环可能显得重复** — 需要每个节日有独特感

### Technical Risks

- **小程序包体限制** — 需要严格控制资源大小，使用分包和远程加载
- **云存档同步** — 需要处理离线/在线状态切换
- **不同设备性能差异** — 需要适配低端机型

### Market Risks

- **小程序游戏竞争激烈** — 需要独特的卖点和有效的推广
- **中国文化游戏的小众性** — 但微信用户基数大，可触达面广
- **内购转化率不确定** — 需要数据分析优化

### Scope Risks

- **内容量超过独立开发能力** — 需要严格控制范围
- **小程序限制可能影响设计** — 需要提前验证技术可行性

### Open Questions

- **体力和时间系统的具体数值** — 需要原型测试
- **内购价格和转化预期** — 需要市场调研
- **iOS原生版的移植方案** — Cocos Creator可跨平台，但需要适配

## MVP Definition

**Core hypothesis**: 玩家会觉得"准备节日 + 与村民互动 + 制作手工艺"这个循环有趣，愿意每天回来玩，并有意愿为装饰付费。

**Required for MVP**:
1. 1个完整节日（中秋）— 准备期 + 庆典 + 节后变化
2. 3-5个村民 — 基础对话 + 友情互动
3. 1种手工艺（做月饼）— 触屏迷你游戏
4. 1个探索区域（村中心 + 1个周边地点）
5. 基础的采集、对话、体力系统
6. 微信登录 + 云存档
7. 2-3个内购项目（装饰、体力）
8. 1个激励视频点位

**Explicitly NOT in MVP**:
- 其他3个节日
- 完整的8-10个村民
- 好友互访社交系统
- 完整内购商店
- 深度剧情线

### Scope Tiers

| Tier | Content | Features | Timeline |
| ---- | ---- | ---- | ---- |
| **MVP** | 1个节日、3-5村民、1手工艺 | 核心循环 + 微信登录 + 基础内购 | 3-4周 |
| **Alpha** | 2个节日、6-8村民、3手工艺 | + 好友社交 + 完整内购 | 8-10周 |
| **Beta (小程序首发)** | 4个节日、8-10村民、4-6手工艺 | 完整功能 | 3-4个月 |
| **iOS原生版** | 同上 + 优化 | 更好的性能和体验 | 后续2-3个月 |
