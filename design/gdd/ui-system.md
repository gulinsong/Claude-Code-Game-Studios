# UI系统 (UI System)

> **Status**: Approved
> **Author**: User + Claude
> **Last Updated**: 2026-04-04
> **Implements Pillar**: 策略简洁

## Overview

UI系统负责管理游戏中所有玩家可见的界面元素，包括 7 个主要屏幕（主菜单、世界选择、关卡选择、游戏 HUD、暂停菜单、胜利结果、失败结果）、屏幕间导航逻辑、以及 UI 动画和反馈。设计分辨率 750x1334（竖屏，FIXED_WIDTH 策略），所有 UI 元素基于锚点布局以适配不同屏幕尺寸。UI 系统从其他系统获取数据（关卡进度、星星数、游戏状态），负责渲染和交互，不包含业务逻辑。对于玩家来说，UI 是**游戏的门面**——清晰的信息层次、流畅的转场、及时的状态反馈决定了玩家是否愿意继续游玩。

## Player Fantasy

玩家应该感到界面是**透明且不碍事的**——在游戏进行时，HUD 提供关键信息但不遮挡游戏区域；在菜单导航时，每个操作都有即时反馈，没有等待加载的空白时刻。

玩家应该感到**进度是可见的**——世界选择界面的星星进度条让玩家知道距离下一个世界还有多远，关卡选择界面的星级显示让玩家知道哪些关值得重玩。

玩家应该感到**操作是直觉的**——暂停、重试、返回的按钮位置一致且可预测；结果弹窗中的星星动画给予即时的情感反馈。

**关键体验目标**：
- **信息清晰**：玩家始终知道自己的状态（第几关、收集几个、还剩几条线）
- **操作流畅**：所有转场 < 0.3 秒，无空白加载画面
- **情感一致**：胜利时的庆祝感、失败时的鼓励感、探索时的期待感

**参考游戏**：
- **《割绳子》**：简洁的关卡选择 + 世界地图 + 流畅的弹窗动画
- **《纪念碑谷》**：极简 UI，不干扰游戏体验

## Detailed Design

### 屏幕架构

游戏包含 7 个 UI 屏幕，分为 3 种类型：

**场景屏幕**（通过场景管理器切换）：
1. 主菜单 (MainMenu)
2. 世界选择 (WorldSelect)
3. 关卡选择 (LevelSelect)
4. 游戏场景 (Gameplay)

**覆盖层**（在游戏场景内渲染，无场景切换）：
5. 暂停菜单 (PauseOverlay)
6. 胜利结果 (WinResultOverlay)
7. 失败结果 (LoseResultOverlay)

### 场景流程

```
主菜单 → 世界选择 → 关卡选择 → 游戏场景
                                    ├── 暂停覆盖层（暂停按钮触发）
                                    ├── 胜利覆盖层（VICTORY 状态触发）
                                    └── 失败覆盖层（DEFEAT 状态触发）
```

**导航规则**：
- 每个屏幕左上角有返回按钮（主菜单除外）
- 游戏场景的返回按钮触发暂停菜单（不是直接返回）
- 暂停菜单的"退出"按钮回到关卡选择（需要确认）
- 结果覆盖层的"下一关"按钮直接加载下一关（如果可用）

### 屏幕 4：游戏 HUD (Gameplay HUD)

**布局**（浮动元素，不使用顶部栏）：

```
┌────────────────────────┐
│ 线段: 2/3  ●●○         │  左上角，y: 1270
│                ⏸        │  右上角，暂停按钮
│                        │
│     [游戏区域]          │
│     全屏可用           │
│                        │
│ 光点: 3/5              │  左下角，y: 64
│                        │
└────────────────────────┘
```

**HUD 元素规格**：

| 元素 | 位置 | 尺寸 | 更新时机 |
|------|------|------|----------|
| 线段计数 | 左上 (30, 1270) | 自动大小 | 每次画线/清除 |
| 光点计数 | 左下 (30, 64) | 自动大小 | 每次收集 |
| 暂停按钮 | 右上 (710, 1270) | 60x60 | 静态 |

**设计决策**：
- 使用浮动元素而非顶部栏，最大化游戏可见区域
- 球在 y=0.85（归一化）发射，左上角 HUD 不会干扰
- 线段计数用实心/空心圆点表示（直觉的剩余资源指示）
- HUD 元素半透明背景（rgba(0,0,0,0.3)），确保在任何游戏背景上可读

**HUD 动画**：
- 线段使用时：对应圆点缩放消失（0.2s）
- 光点收集时：数字跳动（scale 1.0→1.2→1.0，0.15s）
- 暂停按钮：按下缩放 0.9

### 屏幕 5：暂停菜单 (PauseOverlay)

**实现方式**：游戏场景内的覆盖层，无场景切换。物理暂停，游戏状态进入 PAUSED。

**布局**：

```

│    黑色遮罩 alpha=0.6   │
│                        │
│   ┌──────────────┐     │
│   │    暂停       │     │  标题
│   │              │     │
│   │  ┌────────┐  │     │
│   │  │  继续    │  │     │  按钮 1
│   │  └────────┘  │     │
│   │  ┌────────┐  │     │
│   │  │  重试    │  │     │  按钮 2
│   │  └────────┘  │     │
│   │  ┌────────┐  │     │
│   │  │  退出    │  │     │  按钮 3（需确认）
│   │  └────────┘  │     │
│   └──────────────┘     │
│                        │
└────────────────────────┘
└────────────────────────┘
```

**行为**：
- 游戏物理完全停止（`physics.enabled = false`）
- 覆盖层淡入 0.2s
- "继续"按钮恢复物理，覆盖层淡出 0.2s
- "重试"按钮重置当前关卡（重新加载配置）
- "退出"按钮弹出确认对话框："确定退出？当前进度不会保存"
- 确认后返回关卡选择

**触发方式**：
- 点击 HUD 暂停按钮
- 系统返回键（Android/微信）

## Formulas

### 布局计算

```
designWidth = 750
designHeight = 1334
anchorPolicy = FIXED_WIDTH
```

所有 UI 元素使用 Cocos Creator 的 Widget 组件锚定到安全区域边缘。`top`、`bottom`、`left`、`right` 值以设计分辨率为基准。

### 关卡卡片位置

```
cardWidth = 160
cardHeight = 160
columns = 4
horizontalSpacing = 20
verticalSpacing = 30

gridStartX = (designWidth - (columns * cardWidth + (columns-1) * horizontalSpacing)) / 2 + cardWidth/2
gridStartY = designHeight - 200  // 200px 预留给顶部栏

cardPosition(col, row):
    x = gridStartX + col * (cardWidth + horizontalSpacing)
    y = gridStartY - row * (cardHeight + verticalSpacing)
```

| 变量 | 类型 | 范围 | 描述 |
|------|------|------|------|
| col | number | 0-3 | 列索引 |
| row | number | 0-1 | 行索引（8 关 = 2 行 x 4 列） |
| x | number | px | 卡片中心 X |
| y | number | px | 卡片中心 Y |

### 世界卡片位置

```
cardWidth = 650
cardHeight = 200
verticalSpacing = 20

worldCardPosition(index):
    x = designWidth / 2
    y = designHeight - 160 - index * (cardHeight + verticalSpacing) - cardHeight / 2
```

### 星星显示位置（结果覆盖层）

```
starSize = 80  // 每颗星的直径
starSpacing = 30  // 星星间距

starsCenterX = panelWidth / 2
starsStartX = starsCenterX - (3 * starSize + 2 * starSpacing) / 2 + starSize / 2

starPosition(index):  // index 0, 1, 2
    x = starsStartX + index * (starSize + starSpacing)
    y = panelHeight * 0.6  // 面板上部 60% 位置
```

## Edge Cases

| 场景 | 预期行为 | 理由 |
|------|----------|------|
| **最后一关胜利** | "下一关"按钮不显示，只显示"重玩"和"返回" | 无下一关可加载 |
| **世界 4 最后一关胜利** | 显示"恭喜通关！全部关卡完成"特殊文案 | 游戏完成里程碑 |
| **当前世界只有 1 关解锁** | 关卡选择只显示 1 个可点击卡片，其余灰色 | 正常的锁定状态 |
| **快速连续点击关卡卡片** | 防抖 0.3s，忽略重复点击 | 防止重复加载 |
| **暂停期间收到来电/切换应用** | 微信小游戏自动暂停，返回时恢复暂停菜单 | 平台行为 |
| **屏幕旋转** | 不处理，锁定竖屏 | 游戏设计为竖屏 |
| **"继续游戏"但存档损坏** | 隐藏"继续"按钮，只显示"开始游戏" | 优雅降级 |
| **HUD 元素与球路径重叠** | HUD 在安全区域，不与归一化游戏区域重叠 | 设计分离 |
| **所有世界解锁** | 世界选择无锁定卡片，所有卡片可点击 | 正常完成状态 |
| **转场过程中按返回** | 忽略输入直到转场完成 | 防止导航混乱 |
| **结果覆盖层出现时立即点击** | 按钮在动画完成前不可交互 | 确保玩家看到星星动画 |
| **关卡选择界面的返回目标** | 返回世界选择（不是主菜单） | 导航层次 |
| **游戏场景的返回按钮** | 触发暂停菜单（不是直接退出） | 防止误操作 |

## Tuning Knobs

| 参数 | 当前值 | 安全范围 | 增加效果 | 减少效果 |
|------|--------|----------|----------|----------|
| TRANSITION_DURATION | 0.3s | 0.2-0.5s | 转场更慢 | 转场更快速 |
| OVERLAY_FADE_DURATION | 0.2s | 0.1-0.3s | 覆盖层出现更慢 | 覆盖层出现更快 |
| BUTTON_SCALE_ON_PRESS | 0.95 | 0.9-1.0 | 按压反馈更明显 | 按压反馈更微弱 |
| CARD_SCALE_ON_PRESS | 0.97 | 0.95-1.0 | 卡片点击反馈更明显 | 卡片点击反馈更微弱 |
| RESULT_PANEL_WIDTH | 600px | 500-650 | 结果面板更宽 | 结果面板更窄 |
| STAR_SIZE | 80px | 60-100 | 星星更大 | 星星更小 |
| STAR_ANIM_DELAY | 0.4s | 0.2-0.8s | 动画更戏剧化 | 动画更快速 |
| DEBOUNCE_TIME | 0.3s | 0.1-0.5s | 防抖更严格 | 响应更快 |
| BUTTON_DEACTIVATE_DELAY | 1.6s | 1.0-2.5s | 按钮出现更晚 | 按钮出现更早 |

**高风险调优警告**：
- **STAR_ANIM_DELAY** 直接影响结果屏幕的情感节奏——太快会感觉廉价，太慢会让玩家等得不耐烦
- **BUTTON_DEACTIVATE_DELAY** 必须在所有星星动画完成后才能显示按钮，否则玩家会错过动画

## Acceptance Criteria

### 屏幕导航验收

- [ ] 主菜单"开始游戏" → 世界选择场景
- [ ] 世界选择卡片点击 → 关卡选择场景（传入世界 ID）
- [ ] 关卡选择卡片点击 → 游戏场景（传入关卡 ID）
- [ ] 所有关卡选择返回 → 世界选择
- [ ] 所有世界选择返回 → 主菜单
- [ ] 转场动画在 0.3s 内完成

### HUD 验收

- [ ] 线段计数实时更新（画线 +1，达到上限后显示"满"）
- [ ] 光点计数实时更新（收集 +1）
- [ ] 暂停按钮可点击，触发暂停覆盖层
- [ ] HUD 不遮挡游戏区域（归一化 0-1 范围内无 HUD）
- [ ] HUD 在暗色和亮色背景上均清晰可读

### 暂停菜单验收

- [ ] 物理在暂停期间完全停止
- [ ] "继续"恢复物理和游戏
- [ ] "重试"重新加载当前关卡
- [ ] "退出"需要确认对话框
- [ ] 系统返回键触发暂停（非退出）

### 胜利结果验收

- [ ] 星星动画按顺序播放（0.4s 间隔）
- [ ] 按钮在动画完成后出现
- [ ] "下一关"仅当下一关可用时显示
- [ ] "新世界已解锁"横幅在适当时机显示
- [ ] 分数和时间正确显示

### 失败结果验收

- [ ] 显示收集的光点数
- [ ] 如果达到 1-2 星阈值，显示获得的星星
- [ ] 鼓励文案随机轮换
- [ ] "重试"为主按钮（视觉突出）

### 世界选择验收

- [ ] 已解锁世界显示星星进度
- [ ] 锁定世界显示所需星星和进度条
- [ ] 全 3 星世界显示"完美"标签
- [ ] 累计星星数正确显示

### 关卡选择验收

- [ ] 8 个关卡卡片正确排列（2 行 x 4 列）
- [ ] 已完成关卡显示最佳星星数
- [ ] 新解锁关卡显示 "NEW" 标签
- [ ] 锁定关卡灰色不可点击
- [ ] 世界名称和星星总数正确显示

### 性能验收

- [ ] 场景切换 < 300ms
- [ ] HUD 更新 < 1ms（无 GC 压力）
- [ ] 覆盖层打开/关闭 < 200ms
- [ ] 无布局抖动（稳定的帧率）
