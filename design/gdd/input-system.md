# 输入系统 (Input System)

> **Status**: Approved
> **Author**: User + Claude
> **Last Updated**: 2026-03-27
> **Implements Pillar**: 策略简洁

## Overview

输入系统负责处理玩家的触摸操作，将原始触摸事件转换为游戏可用的输入数据。玩家通过手指在屏幕上滑动来画线，输入系统追踪触摸路径并输出线段数据给画线反弹系统。这个系统是游戏交互的基础，没有它玩家无法与游戏互动。

## Player Fantasy

玩家应该感觉画线是**流畅自然的延伸**——手指到哪，线就跟到哪，没有明显延迟或卡顿。操作应该像用手指在沙滩上画画一样直觉，不需要学习复杂的手势。画线是策略的执行，不是技巧的考验；输入系统不应该成为玩家通关的障碍。

## Detailed Design

### Core Rules

1. **触摸检测**：监听屏幕触摸事件，只处理第一个触摸点（单指操作）
2. **线段生成**：触摸结束时，生成一条从起点到终点的直线
3. **实时预览**：触摸移动过程中，实时显示预览线（半透明效果）
4. **线段确认**：触摸结束时，预览线变为确认线（不透明）
5. **撤销功能**：玩家可以点击已确认的线段将其删除，恢复线段配额
6. **配额限制**：每关最多画3条线（配额由游戏状态管理提供）
7. **时间限制**：无时间限制，玩家可以随时画线（球发射前后都可以）

### States and Transitions

| State | Entry Condition | Exit Condition | Behavior |
|-------|----------------|----------------|----------|
| **Idle** | 游戏开始 / 线画完 / 线撤销 | 触摸开始 | 等待玩家输入 |
| **Drawing** | 触摸开始 | 触摸结束 | 实时显示预览线，跟随手指移动 |
| **LinePlaced** | 触摸结束 | 球撞到线 / 玩家撤销 | 显示确认线，等待物理交互或撤销 |

**状态转换图**：
```
Idle --[touch start]--> Drawing --[touch end]--> LinePlaced
  ↑                                                 |
  +-----------------[player undo]-------------------+
```

### Interactions with Other Systems

| System | Direction | Interface | Description |
|--------|-----------|-----------|-------------|
| **画线反弹系统** | 输出 → | `onLineCreated(start: Vec2, end: Vec2)` | 确认线段时，发送起点终点坐标 |
| **画线反弹系统** | 输出 → | `onLineRemoved(lineId: string)` | 撤销线段时，发送线段ID |
| **游戏状态管理** | 输入 ← | `canDrawLine(): boolean` | 查询是否还能画线（配额检查） |
| **游戏状态管理** | 输出 → | `onLineCountChanged(count: number)` | 线段数量变化时通知 |
| **视觉反馈系统** | 输出 → | `showPreviewLine(start: Vec2, end: Vec2)` | 实时显示预览线 |
| **视觉反馈系统** | 输出 → | `hidePreviewLine()` | 触摸结束时隐藏预览 |
| **音频系统** | 输出 → | `playLinePlaceSound()` | 线段确认时播放音效 |
| **音频系统** | 输出 → | `playLineUndoSound()` | 撤销线段时播放音效 |

## Formulas

### 线段长度计算

```
lineLength = sqrt((endX - startX)² + (endY - startY)²)
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| startX, startY | Vec2 | 0 - 屏幕尺寸 | 触摸事件 | 触摸起点坐标（像素） |
| endX, endY | Vec2 | 0 - 屏幕尺寸 | 触摸事件 | 触摸终点坐标（像素） |
| lineLength | number | 0 - 屏幕对角线 | 计算得出 | 线段长度（像素） |

**Expected output range**: 20 - 屏幕对角线长度

### 最小线段验证

```
if (lineLength < MIN_LINE_LENGTH) {
    // 线段太短，不创建，忽略这次画线
    return;
}
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| MIN_LINE_LENGTH | number | 20 | 配置常量 | 最小线段长度，防止误触 |

**Edge case**: 当线段长度 < 20 像素时，不创建线段，不消耗配额

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| **线段太短 (< 20px)** | 忽略这次画线，不创建线段，不消耗配额 | 防止误触创建无效线段 |
| **触摸被中断**（来电、切换应用） | 取消当前画线操作，不创建线段 | 避免创建意外的线段 |
| **球撞到线后尝试撤销** | 拒绝撤销，线段已激活不能删除 | 防止玩家利用撤销机制作弊 |
| **超出配额时尝试画线** | 不进入 Drawing 状态，显示"线段用完"提示 | 明确告知玩家限制 |
| **触摸点在 UI 区域** | 不触发画线，让 UI 系统处理该触摸 | 避免与按钮点击冲突 |
| **快速连续点击** | 只处理第一个触摸点，忽略后续 | 单指操作，防止多指干扰 |
| **画线时手指滑出屏幕** | 以屏幕边界为终点创建线段 | 保留玩家意图 |
| **暂停时触摸屏幕** | 不响应画线，暂停状态屏蔽输入 | 暂停时不应影响游戏 |

## Dependencies

| System | Direction | Nature of Dependency | Status |
|--------|-----------|---------------------|--------|
| **画线反弹系统** | 输出 → | 接收线段坐标数据，创建物理挡板 | 未设计 |
| **游戏状态管理** | 双向 | 查询配额 + 通知线段数量变化 | 未设计 |
| **视觉反馈系统** | 输出 → | 发送预览线/确认线的显示指令 | 未设计 |
| **音频系统** | 输出 → | 触发音效播放 | 未设计 |

**注意**：输入系统是 Foundation 层，没有上游依赖。所有依赖它的系统都在 Feature 或 Presentation 层。

## Tuning Knobs

| Parameter | Current Value | Safe Range | Effect of Increase | Effect of Decrease |
|-----------|--------------|------------|-------------------|-------------------|
| **MIN_LINE_LENGTH** | 20 px | 10-50 px | 减少误触，但可能忽略玩家意图 | 允许更精细的短线，但容易误触 |
| **PREVIEW_LINE_OPACITY** | 0.5 | 0.3-0.7 | 预览更明显 | 预览更含蓄 |
| **MAX_LINES_PER_LEVEL** | 3 | 1-5 | 策略性降低，解法更多样 | 策略性提高，难度增加 |
| **TOUCH_SAMPLE_RATE** | 60 Hz | 30-120 Hz | 预览更流畅 | 性能更好，但预览可能卡顿 |

**交互影响**：
- `MAX_LINES_PER_LEVEL` 由游戏状态管理持有，输入系统只是查询
- `TOUCH_SAMPLE_RATE` 影响预览线的流畅度，但不影响最终线段（只用起终点）

## Visual/Audio Requirements

| Event | Visual Feedback | Audio Feedback | Priority |
|-------|----------------|---------------|----------|
| 触摸开始 | 显示预览线起点（小圆点） | 无 | Medium |
| 触摸移动 | 预览线跟随手指延伸 | 无 | High |
| 触摸结束（线段有效） | 预览线变为确认线（实线） | 播放"放置音效" | High |
| 触摸结束（线段太短） | 预览线消失，显示抖动动画 | 播放"错误提示音" | Medium |
| 撤销线段 | 线段淡出消失 | 播放"撤销音效" | Medium |
| 超出配额尝试画线 | 显示"线段用完"提示 | 播放"禁止音效" | Low |

## UI Requirements

| Information | Display Location | Update Frequency | Condition |
|-------------|-----------------|-----------------|-----------|
| 剩余线段数 | 屏幕左上角 | 线段数量变化时 | 始终显示 |
| 线段用完提示 | 屏幕中央 | 即时 | 配额用完时尝试画线 |

## Acceptance Criteria

- [ ] 玩家触摸屏幕并滑动时，显示半透明预览线
- [ ] 玩家松手时，预览线变为确认线（不透明）
- [ ] 线段长度 < 20px 时，不创建线段，显示抖动动画
- [ ] 线段数量达到上限时，无法继续画线，显示提示
- [ ] 玩家点击已确认的线段可以撤销（球未撞到前）
- [ ] 球撞到线段后，该线段无法撤销
- [ ] 多指触摸时，只处理第一个触摸点
- [ ] 触摸 UI 区域不触发画线
- [ ] 性能：触摸响应延迟 < 16ms（60fps）
- [ ] 无硬编码值：所有参数可通过配置文件调整
