# 手工艺系统原型

> **核心问题**: 迷你游戏的操作手感是否有趣？玩家是否能在短时间内理解操作？
> **创建日期**: 2026-03-24

---

## 如念

验证《岁时记》手工艺系统的核心假设：
- 猏你游戏操作是否有趣？
- 玩家是否能快速理解操作？
- 操作反馈是否足够？

## 吺于设计文档的假设

根据 `design/gdd/crafting-system.md`，手工艺采用**配方驱动 + 迷你游戏**模式：
- 选择食谱 → 确认材料 → 进入迷你游戏 → 完成制作
- 迷你游戏有多个阶段（揉面、包馅、压模等）
- 操作类型：点击节奏、拖拽、滑动
- 目标时长：10-20 秒

## 原型范围

**包含**:
- 单个迷你游戏：做月饼（3 个阶段）
- 简单的 UI 显示当前阶段和操作提示
- 基本的进度反馈

**不包含**:
- 完整的制作流程（选择食谱、确认材料）
- 体力消耗
- 背包系统
- 熟练度系统
- 高品质产出

---

## 运行说明

1. 打开 `index.html` 在浏览器中
2. 点击屏幕开始游戏
3. 按照提示完成 3 个阶段
4. 观察操作感受和反馈

---

## 测试检查清单

- [ ] 阶段 1（点击节奏）是否有节奏感？
- [ ] 阶段 2（拖拽）是否直观？
- [ ] 阶段 3（点击选择）是否有选择感？
- [ ] 整体时长是否在 10-20 秒范围内？
- [ ] 操作反馈是否清晰？
- [ ] 是否想要重复玩？
