# 配置系统 (Config System)

> **Status**: Designed
> **Author**: User + Claude
> **Last Updated**: 2026-03-24
> **Implements Pillar**: 基础设施 — 所有系统的基础

## Overview

配置系统是《岁时记》的**数据定义中心**，负责加载、解析和提供所有静态游戏数据。它管理的配置类型包括：材料定义、食谱定义、村民定义、节日定义、平衡参数（体力、时间）等。

配置系统确保策划可以在**不修改代码**的情况下调整游戏内容，通过 JSON 配置文件定义所有游戏数据。系统支持本地配置（随包体发布）和远程配置（CDN 热更新），以适应小程序包体限制。

玩家**不直接与配置系统交互**——这是一个后台基础设施。玩家体验到的"丰富的内容"和"平衡的数值"是这个系统在后台支撑的。其他系统（背包、手工艺、时间、体力等）通过配置系统获取数据定义，实现数据与逻辑的分离。

## Player Fantasy

此系统是基础设施，玩家不直接感知。但配置系统的设计质量会**间接影响玩家体验**：

**设计良好时玩家感受到的**：
- **丰富的内容感**：每个材料、食谱、村民都有独特的名字、描述和背景故事
- **平衡的节奏**：体力消耗、制作时间、材料获取难度相互配合，形成舒适的游戏节奏
- **发现惊喜**：配置中隐藏的稀有材料、特殊对话触发条件、节日限定内容

**设计不佳时玩家感受到的**：
- **内容空洞**：材料只是"物品A"、"物品B"，没有文化内涵
- **节奏失衡**：某项活动耗时过长或过短，体力消耗不合理
- **重复感**：不同节日/村民的配置过于相似，缺乏独特性

此系统主要支撑 **Pillar 4: 发现民间之美** —— 通过配置中嵌入的文化背景和故事描述，让每个收集物都有故事。

## Detailed Design

### Core Rules

1. **配置类型划分**

| 配置类型 | 文件名 | 内容 | 加载时机 |
|---------|--------|------|---------|
| **全局参数** | `global.json` | 体力上限、恢复速率、日长、季节长度 | 启动时 |
| **材料定义** | `materials.json` | 材料ID、名称、图标、稀有度、描述、文化背景 | 启动时 |
| **食谱定义** | `recipes.json` | 食谱ID、输入材料、输出物品、制作时间 | 启动时 |
| **村民定义** | `villagers.json` | 村民ID、名称、头像、喜好、对话ID | 按需 |
| **节日定义** | `festivals.json` | 节日ID、时间、准备期、奖励 | 按需 |
| **对话定义** | `dialogs.json` | 对话ID、文本、选项、条件 | 按需 |

2. **配置文件格式**
   - 使用 JSON 格式存储配置数据
   - 每个配置文件包含一个配置数组或配置对象
   - 配置项必须有唯一的 `id` 字段作为标识符
   - 支持嵌套结构（如食谱的输入材料列表）

3. **加载策略**（考虑小程序包体限制）
   - **启动加载**：全局参数、材料、食谱（核心循环必需，体积小）
   - **按需加载**：村民、节日、对话（内容多，可分包或远程加载）
   - **远程更新**：支持从 CDN 加载最新配置（热更新，无需重新发布）

4. **配置数据访问**
   - 所有配置通过 `ConfigManager` 单例访问
   - 提供 `get{Type}ById(id)` 方法获取单个配置
   - 提供 `getAll{Type}s()` 方法获取某类型所有配置
   - 配置数据只读，运行时不可修改

5. **配置验证**
   - 启动时验证配置完整性和格式正确性
   - 缺失必要字段时记录错误并使用默认值
   - 重复 ID 时记录警告并保留第一个

### States and Transitions

| State | Entry Condition | Exit Condition | Behavior |
|-------|----------------|----------------|----------|
| **Unloaded** | 系统初始化 | 调用 `load()` | 配置数据未加载，所有查询返回空或默认值 |
| **Loading** | 调用 `load()` | 加载完成或失败 | 异步加载配置文件，显示加载进度 |
| **Ready** | 加载完成 | 调用 `reload()` 或系统销毁 | 配置数据可用，响应查询请求 |
| **Error** | 加载失败 | 调用 `retry()` 或使用缓存 | 使用默认值或缓存，记录错误日志 |

**状态转换图**：
```
Unloaded → Loading → Ready
    ↑          ↓
    └──── Error ←┘
         ↓
      retry() → Loading
```

**热更新机制**：
- 远程配置更新时，可调用 `reload()` 重新加载指定配置类型
- 加载期间继续使用旧配置，保证游戏不中断
- 新配置加载成功后平滑切换，发布 `config:reloaded` 事件通知相关系统

### Interactions with Other Systems

| 系统 | 交互方式 | 数据流 | 说明 |
|------|---------|--------|------|
| **材料系统** | 读取配置 | 配置 → 材料 | 获取材料定义（名称、图标、稀有度、描述） |
| **背包系统** | 间接依赖 | 配置 → 材料系统 → 背包 | 通过材料系统获取物品信息 |
| **食谱系统** | 读取配置 | 配置 → 食谱 | 获取食谱定义（输入/输出、制作时间） |
| **手工艺系统** | 读取配置 | 配置 → 食谱系统 → 手工艺 | 通过食谱系统获取制作配方 |
| **时间系统** | 读取配置 | 配置 → 时间 | 获取时间参数（日长、季节长度） |
| **体力系统** | 读取配置 | 配置 → 体力 | 获取体力参数（上限、恢复速率） |
| **对话系统** | 读取配置 | 配置 → 对话 | 获取对话定义（文本、选项、条件） |
| **任务系统** | 间接依赖 | 配置 → 对话系统 → 任务 | 任务目标通过对话系统定义 |
| **节日筹备系统** | 读取配置 | 配置 → 节日 | 获取节日定义（时间、准备期、奖励） |
| **收集系统** | 读取配置 | 配置 → 材料/食谱 | 获取所有可收集物品的定义 |
| **事件系统** | 发布事件 | 配置 → 事件 | 配置加载完成/热更新时发布事件 |

**事件交互**：
- `config:loaded` — 配置加载完成，其他系统可以开始读取
- `config:reloaded` — 配置热更新完成，相关系统应刷新缓存
- `config:error` — 配置加载失败，记录错误并使用默认值

## Formulas

配置系统本身无复杂游戏公式，但有以下性能相关计算：

### 内存占用估算

```
memory_usage = sum(config_file_size) + index_overhead
index_overhead ≈ config_count × 100 bytes
```

| Variable | Type | Range | Source | Description |
|----------|------|-------|--------|-------------|
| config_file_size | int | 1-50KB | asset | 配置文件大小 |
| load_speed | int | ~1MB/s | device | 小程序本地读取速度 |
| parse_time | int | 1-10ms | runtime | JSON 解析时间 |

**Expected output range**: < 100ms（单文件），< 500ms（全部启动配置）

### 配置文件大小预算

| 配置类型 | 预估条目数 | 单条大小 | 文件大小预算 |
|---------|-----------|---------|-------------|
| 全局参数 | ~20 | 50B | < 2KB |
| 材料定义 | ~50 | 200B | < 15KB |
| 食谱定义 | ~30 | 300B | < 12KB |
| 村民定义 | ~10 | 500B | < 8KB |
| 节日定义 | ~4 | 1KB | < 6KB |
| 对话定义 | ~100 | 300B | < 40KB |
| **总计** | — | — | **< 100KB** |

## Edge Cases

| Scenario | Expected Behavior | Rationale |
|----------|------------------|-----------|
| 配置文件不存在 | 使用内置默认配置，记录警告 | 保证游戏能启动，后续可通过热更新获取正确配置 |
| 配置项 ID 不存在 | 返回 `undefined`，记录警告 | 调用方应处理 undefined 情况 |
| 重复 ID | 保留第一个，记录警告 | 避免数据覆盖导致的意外行为 |
| 必要字段缺失 | 使用字段默认值（在配置定义中声明），记录警告 | 优雅降级，不阻塞游戏 |
| 配置文件过大 (>100KB) | 记录警告，继续加载 | 不阻塞，但提醒策划优化 |
| 热更新失败 | 继续使用旧配置，记录错误 | 保证游戏不中断 |
| 远程加载超时 | 使用本地缓存配置，记录警告 | 离线场景可用 |
| JSON 解析失败 | 抛出错误，阻止游戏启动 | 配置格式错误是致命问题，需在开发阶段修复 |

| Parameter | Current Value | Safe Range | Effect of Increase | Effect of decrease |

## Implementation Notes

### 推荐实现方式 (Cocos Creator / TypeScript)

```typescript
// 简化示例 - 完整实现在代码阶段
interface MaterialConfig {
    id: string;
    name: string;
    icon: string;
    rarity: 'common' | 'rare' | 'legendary';
    description: string;
}

interface RecipeConfig {
    id: string;
    inputs: { materialId: string; count: number }[];
    output: { itemId: string; count: number };
    craftTime: number; // 秒
}

class ConfigManager {
    private materials: Map<string, MaterialConfig> = new Map();
    private recipes: Map<string, RecipeConfig> = new Map();

    async load(): Promise<void>;
    getMaterialById(id: string): MaterialConfig | undefined;
    getRecipeById(id: string): RecipeConfig | undefined;
    getAllMaterials(): MaterialConfig[];
}
```
